초기에는 컴퓨터에 대한 지식이 부족했고, 단순 사용에 그쳐 왔습니다. 그러나 반자동화 기계를 다루면서 발생한 문제들을 해결하며 컴퓨터와 프로그래밍에 대한 호기심이 자라게 되었습니다. 시간이 흘러갈수록 더 심층적으로 공부하고자 결심하고, 현재는 전문적인 교육을 받으며 웹 개발에 대한 기술력을 키우고 있습니다.

실제 프로젝트를 통해 개발에 대한 열정을 실천해 왔습니다. 특히 최근에는 웹 개발 프로젝트에서 백엔드를 주로 다루었으며, 데이터베이스 관리와 API 개발을 담당했습니다. 클라이언트와의 원활한 의사소통을 통해 프로젝트를 성공적으로 완료하였으며, 이를 통해 다양한 기술 스택을 습득하고 응용할 수 있는 능력을 키웠습니다.

트러블 슈팅

  • 사진 용량 및 비용 관리 문제로 인해 DB에 저장하는 대신 Cloudinary를 도입했습니다. AWS는 비용 문제로 제외했습니다. 이로써 사진 관리에 대한 우리의 노력이 비용 효율성과 시스템 성능 향상에 이바지했습니다.

  • 클라이언트에서 서버에 API 요청을 보냈을 때 CORS 에러가 발생했습니다. HTTP 메소드를 변경해봤지만 문제가 해결되지 않아, 보안상의 이유를 고려하여 서버의 CORS 설정을 일시적으로 전부 허용으로 변경했습니다. 이후 CORS 허용 헤더를 추가하여 클라이언트에서의 API 요청을 허용하도록 설정했고, CORS 에러가 사라졌습니다. 에러가 사라진 것을 확인한 후, 서버의 CORS 설정을 다시 적절한 범위로 수정하여 보안을 강화했습니다.

  • 찜하기 기능을 구현하는 과정에서 DB에 너무 많은 찜/해제 요청이 발생할 경우 시스템에 부하가 발생할 것이 우려되었습니다. 이 문제를 해결하기 위해 해당 페이지의 프론트엔드와 논의를 진행하였고, 다양한 의견이 주고받아졌습니다. 최종적으로, 페이지가 변경되기 전에 찜하기 요청을 보내는 방식이 가장 효율적이라는 결론에 도달하였습니다.
아키텍쳐
  • 백엔드 API 서버로는 Spring Boot를 이용하였으며, 데이터베이스는 MySQL을 사용하였습니다.

  • 코드의 정확성을 확인하기 위해 Swagger를 활용하여 테스트를 진행했습니다.
